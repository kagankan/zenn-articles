---
title: "リダイレクト・自動遷移"
---

ユーザーの明示的な操作（リンクやボタンのクリック）を介さずに、別の URL・ページへ移動させる処理です。別 URL への転送（リダイレクト）のほか、ログイン後の遷移、時間経過での自動遷移、入力内容に応じた画面切り替えなどが含まれます。便利な反面、ユーザーの意図しないタイミングで画面が切り替わると、混乱や操作ミスの原因になります。

## 使用の前に✋

- **その遷移は、ユーザーが意図したものでしょうか？**
  - 操作や予告なく画面が切り替わると、特にスクリーンリーダー利用者やロービジョンのユーザーは「今どこにいるのか」を見失います。
  - 自動で移動させる必要が本当にあるかを検討してください。リンクやボタンを置いて、ユーザー自身に移動を選んでもらえる場面も多いはずです。

## 実装するときに気をつけること

### アクセシビリティのために

- [ ] **時間制限つきの自動リダイレクトは、止められる・延ばせる**ようにする
  - **なぜ**: 「5 秒後に移動します」のようなタイマー遷移は、読むのが遅い人・操作に時間がかかる人にとって、読み終える前にページが切り替わってしまいます。
  - **確認方法**: 遅延つきで自動遷移させる場合、解除・延長・即時移動のいずれかの手段があるか。そもそも遅延が不要なら、サーバー側の HTTP リダイレクト（301 / 302 / 307 など）で即座に転送すれば、この問題自体が発生しません。
  - **該当する達成基準**: [達成基準 2.2.1 タイミング調整可能 (レベル A)](https://waic.jp/translations/WCAG21/Understanding/timing-adjustable.html)
- [ ] **入力・選択しただけで勝手に画面遷移しない**ようにする
  - **なぜ**: たとえば `select` を選んだ瞬間にページ遷移すると、キーボードで選択肢を順に送っているユーザーは、目的の項目に到達する前に意図せず遷移してしまいます。
  - **確認方法**: フォーム部品の値を変えただけで遷移していないか。遷移には「移動する」ボタンなど明示的な操作を用意するか、事前にその動作を予告しているか。
  - **該当する達成基準**: [達成基準 3.2.2 入力時 (レベル A)](https://waic.jp/translations/WCAG21/Understanding/on-input.html)
- [ ] **遷移したことと、遷移先がどこかが伝わる**ようにする
  - **なぜ**: ページが切り替わったこと・どこに来たのかが伝わらないと、特に画面を見ないユーザーは現在地を見失います。
  - **確認方法**: 遷移後、`title` 要素が遷移先の内容を表しているか。SPA などページ全体を再読み込みしない遷移では、フォーカスを適切な位置（見出しなど）へ移し、変化を伝える。
  - **該当する達成基準**: [達成基準 2.4.2 ページタイトル (レベル A)](https://waic.jp/translations/WCAG21/Understanding/page-titled.html) ／ [達成基準 3.2.5 要求による変化 (レベル AAA)](https://waic.jp/translations/WCAG21/Understanding/change-on-request.html)

### その他、気にしておきたいこと

- [ ] ブラウザの「戻る」が壊れないか。`meta refresh` やクライアント側の遷移では、戻るボタンで再びリダイレクトされてループする・前のページに戻れない、といった事故が起きがちです。可能な限りサーバー側の HTTP リダイレクトを優先してください。

## ありがちな失敗例

### `meta refresh` で時間制限つきリダイレクトしている

### `select` の `change` で即ページ遷移している

### SPA の画面遷移でフォーカスが置き去りになる

## 実装例

### 時間制限つきの自動リダイレクト

```html:NG
<!-- 5秒後に自動遷移。ユーザーは止められず、戻るボタンも壊れやすい -->
<meta http-equiv="refresh" content="5; url=/new-page" />
```

遅延が不要なら、マークアップを介さずサーバー側で転送します（例: HTTP ステータス 301 / 302 / 307 と `Location` ヘッダー）。タイマーが存在しないため、2.2.1 の問題は生じません。

どうしてもページを表示してから案内する場合は、自動遷移させず、ユーザーが押せるリンクを置きます。

```html:OK
<p>ページは移動しました。</p>
<p><a href="/new-page">新しいページへ移動する</a></p>
```

### 入力に応じた遷移

```html:NG
<!-- 選択した瞬間に遷移してしまう（3.2.2 違反） -->
<select onchange="location.href = this.value">
  <option value="/a">ページA</option>
  <option value="/b">ページB</option>
</select>
```

選択は「値を決める」操作にとどめ、遷移は明示的な操作（送信ボタンなど）に委ねます。

```html:OK
<form action="/go" method="get">
  <label>移動先：
    <select name="dest">
      <option value="/a">ページA</option>
      <option value="/b">ページB</option>
    </select>
  </label>
  <button type="submit">移動する</button>
</form>
```
