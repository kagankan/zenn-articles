## アコーディオンとは

アコーディオンは、開閉することでコンテンツを表示・非表示にする UI パターンです。最初に表示する要素を見出しのみに減らし、ユーザーが必要に応じて表示できます。
最初に表示する情報量を減らせるため、ページ全体を見渡しやすくなります。

## 使用の前に✋

- **アコーディオンでコンテンツを隠すことは適切でしょうか？**
  - わざわざクリックしないと表示されない要素は、ユーザーの操作量を増やしてしまうため面倒にもなります。
  - 流し見するユーザーには、コンテンツの存在すら伝わらないかもしれません。
  - 使用するページや中に入れるコンテンツの性質に照らして、「わざわざクリックしないと表示されないこと」が適切か判断の上で使用してください。

## 実装するときに気をつけること

### アクセシビリティの考慮事項

- [ ] **キーボードだけで開閉できる**ようにする
  - **なぜ**: マウスを使えない状況にある人（手を動かすのが難しいといった肉体的な原因の他にも、一時的にマウスが壊れたなど）はキーボードで操作します。クリックでしか開けないと、その人たちは中のコンテンツにたどり着けません。
  - **確認方法**: マウスを使わず、`Tab` で開閉ボタンにフォーカスでき、`Enter` / `Space` で開閉できるか。
  - **該当する達成基準**: [達成基準 2.1.1 キーボード (レベル A)](https://waic.jp/translations/WCAG21/Understanding/keyboard.html)
- [ ] **フォーカスインジケータが見える**ようにする
  - **なぜ**: キーボード利用者は、いま画面のどこを操作しているかをフォーカスの枠（リング）で判断します。枠が見えないと、どこを操作しているかわからなくなります。
  - **確認方法**: `Tab` で開閉ボタンに移動したとき、枠がはっきり表示されるか。特に角丸のために `overflow: hidden;` を使っていると枠が隠れてしまうことがあります。
  - **該当する達成基準**: [達成基準 2.4.7 フォーカスの可視化 (レベル AA)](https://waic.jp/translations/WCAG21/Understanding/focus-visible.html)
- [ ] **ボタンであることと開閉状態がスクリーンリーダーに伝わる**ようにする
  - **なぜ**: 画面を見ないで利用するスクリーンリーダーユーザーは、要素が「ボタンであること」「いま開いているか閉じているか」を音声で受け取ります。見た目（▼マークなど）だけで表現していると、これらが伝わりません。
  - **確認方法**: スクリーンリーダーで、ボタンとして読み上げられ、かつ開閉状態（展開/折りたたみ）が伝わるか。`details` / `summary` を使うと自然に満たせます。
  - **該当する達成基準**: [達成基準 4.1.2 名前 (name)・役割 (role) 及び値 (value) (レベル A)](https://waic.jp/translations/WCAG21/Understanding/name-role-value.html)
- [ ] **モーションアニメーションを無効にできる**ようにする（開閉アニメーションを使う場合）
  - **なぜ**: 動きで気分が悪くなる人（前庭障害など）がいます。動きを減らす設定をしているユーザーには、開閉アニメーションを止めて見せる必要があります。
  - **確認方法**: OS の「視差効果を減らす」等を有効にした状態で、アニメーションが止まり瞬時に開閉するか。`prefers-reduced-motion` メディアクエリで実現できます。
    - `prefers-reduced-motion` は Chrome DevTools の「レンダリング」タブでエミュレートできます。
    - https://developer.chrome.com/docs/devtools/rendering/emulate-css?hl=ja#emulate_css_media_feature_prefers-color-scheme
  - **該当する達成基準**: [達成基準 2.3.3 インタラクションによるアニメーション (レベル AAA)](https://waic.jp/translations/WCAG21/Understanding/animation-from-interactions.html)
    - なお、この達成基準で無効にできるべきとされているのは「モーションアニメーション」（場所がが変わったりサイズが変わったりする、動きのアニメーション）です。色の変化や透明度変化はこれに含まれません。
- [ ] **文字の拡大・行間の変更・画面幅の縮小でも、見出しや中身が切れたり重ならない**ようにする
  - **なぜ**: ロービジョンのユーザーはブラウザのズームや文字サイズの拡大で読み、行間や文字間隔を広げて読む人もいます。狭い画面で見る人もいます。見出し部分を 1 行・固定の高さで作っていると、文字が増えたり拡大されたときにはみ出して読めなくなります。翻訳や言語の違いで文字量が変わる場合も同様です。
  - **確認方法**: 次のいずれの状況でも、テキストが切れず・重ならず・横スクロールなしで全部読めるか（固定の `height` を避け、`white-space: nowrap` で折り返しを止めないのがポイント）。
    - ブラウザのズームで文字を 200% に拡大したとき
    - 行間をフォントサイズの 1.5 倍に広げたとき (`line-height: 1.5;` を設定する)
    - 画面幅を 320px まで狭めたとき
  - **該当する達成基準**: [達成基準 1.4.4 文字サイズの変更 (レベル AA)](https://waic.jp/translations/WCAG21/Understanding/resize-text.html) ／ [達成基準 1.4.12 テキストの間隔 (レベル AA)](https://waic.jp/translations/WCAG21/Understanding/text-spacing.html) ／ [達成基準 1.4.10 リフロー (レベル AA)](https://waic.jp/translations/WCAG21/Understanding/reflow.html)

### ユーザビリティの考慮事項

WCAG の適合要件ではありませんが、より使いやすくするために考慮するとよい観点です。

- [ ] **開かなくても、中のテキストがページ内検索（Ctrl / ⌘ + F）で見つかる**ようにする
  - **なぜ**: 目的の情報がどこにあるか探すとき、ページ内検索が便利です。特にアコーディオンで隠されている情報は見つけるのが難しくなります。アコーディオンを開かないと中身が検索対象にならないと、そこに情報があっても見つけられず、「無い」と判断されてしまいます。
  - **確認方法**: アコーディオンを閉じた状態で、中のテキストをブラウザのページ内検索で探してヒットするか。`details` / `summary` や `hidden="until-found"` を使うと、閉じていても検索でヒットし、自動的に開きます。
- [ ] **長いコンテンツでも、見出しを画面内に留めて開閉しやすくする**（任意）
  - **なぜ**: コンテンツが画面の高さを超えて長いと、開いたあとに閉じようとしても見出しまでスクロールして戻る必要があり、操作が面倒です。見出しを `position: sticky` でスクロールに追従させると、下まで読んだ位置からでもすぐに閉じられます。
  - **確認方法**: 中身の長い項目を開いて下までスクロールした状態でも、見出し（開閉ボタン）が画面内に表示され、そのまま閉じられるか。
  - **補足**: 角丸のための `overflow` に `hidden` を使うと `sticky` が効かなくなります。`clip` を使うと追従させられます（実装例参照）。

:::details CSS コード

```css:CSS
summary {
  /* summary に対して cursor: pointer を指定する場合、
  コンポーネントに対してではなく、サイト全体の「クリックして開閉可能な要素の共通ルール」として扱うべきあるため、
  class で個別に指定するよりも、グローバルに一括で指定することが望ましいと考えます。 */
  cursor: pointer;
}

.details {
  /* アコーディオン全体に角丸 (border-radius) を適用するとき、中の要素に背景色が設定されてもはみ出さないよう overflow を指定します */
  /* hidden ではなく clip を使用することで、見出しの追従 (position: sticky) を行うことができます。（追従させない場合は hidden でも特に問題ありません） */
  overflow: clip;
  border: 2px solid #666;
  border-radius: 0.5rem;

  /* CSS (interpolate-size) を用いてアニメーションさせる */
  &.details--animate-with-css {
    /* allow-keywords によって、height: auto などキーワードへのアニメーションが適用される */
    interpolate-size: allow-keywords;

    /* ユーザーがアニメーションを無効にしていない場合のみアニメーションを行う */
    @media (prefers-reduced-motion: no-preference) {
      /* ::details-content は details のコンテンツ部分を表す疑似要素です */
      &::details-content {
        /* ::details-content に対する transition を使用する場合、
        はみ出した部分を隠さないとアニメーション中に中身のテキストが見えてしまいます。 */
        /* このサンプルでは、親要素の .details に対して overflow: clip を適用しているため不要です。 */
        /* overflow: clip; */

        /* block-size は横書きコンテンツのみ考慮する場合は height に置き換えても大丈夫です */
        /* height と違い block-size は書字方向に依存したプロパティのため、縦書きでもうまく動作します */
        block-size: 0;
        transition-property: block-size, content-visibility;
        transition-duration: 0.4s;
        /* allow-discrete キーワードによって、content-visibility のような離散プロパティもアニメーションさせることができる */
        transition-behavior: allow-discrete;
      }

      &[open]::details-content {
        block-size: auto;
      }
    }
  }
}

.details__summary {
  /* position: sticky と inset-block-start で見出しをページ上部に追従させる */
  position: sticky;
  /* inset-block-start は、横書きのみ考慮する場合 top に置き換えても大丈夫です */
  inset-block-start: 0;

  display: grid;
  column-gap: 0.5rem;
  justify-content: space-between;
  padding: 1rem;
  font-weight: 700;
  background-color: #ddd;

  &:focus-visible {
    /* フォーカスリングが overflow: hidden/clip; で隠れてしまう問題の対策 */
    outline-offset: -4px;
  }

  @media (any-hover: hover) {
    &:where(summary):hover {
      background-color: #ccc;
    }
  }

  /* 閉じているときは「＋」、開いているときは「－」を表示するアイコン   */
  &::before,
  &::after {
    display: block;
    grid-area: 1 / 2;
    width: 1em;
    height: 2px;
    margin-block: auto;
    content: "";
    background-color: currentcolor;

    /* ユーザーがアニメーションを無効にしていない場合のみアニメーションを行う */
    @media (prefers-reduced-motion: no-preference) {
      transition: transform 0.2s;
    }
  }

  &::after {
    transform: rotate(90deg);
  }

  /* :not(.is-closing) は JS でアニメーションを実装する場合にのみ必要です */
  &:is(details[open]:not(.is-closing) *) {
    &::after {
      transform: rotate(0deg);
    }
  }
}

.details__content {
  padding: 2rem;
}
```

:::details JavaScript コード

```ts:JavaScript
const SELECTOR_DETAILS = "details";
const SELECTOR_SUMMARY = "summary";
const SELECTOR_CONTENT = "[data-details-content]";
const CLASS_BEFORE_CLOSE = "is-closing";

const DURATION = 300;
const EASING = "ease";

const allDetails = document.querySelectorAll(SELECTOR_DETAILS);
allDetails.forEach((details) => {
  const summary = details.querySelector(SELECTOR_SUMMARY);
  const content = details.querySelector(SELECTOR_CONTENT);

  if (!summary || !content) return;

  content.style.overflow = "hidden";

  summary.addEventListener("click", (event) => {
    // ユーザーがアニメーションを無効にしていない場合のみアニメーションを行う
    if (!window.matchMedia("(prefers-reduced-motion: no-preference)").matches) {
      return;
    }

    if (details.open) {
      // 閉じる
      event.preventDefault();
      details.classList.add(CLASS_BEFORE_CLOSE);

      const animation = content.animate(
        { height: [`${content.scrollHeight}px`, "0px"] },
        { duration: DURATION, easing: EASING }
      );

      animation.onfinish = () => {
        details.open = false;
        details.classList.remove(CLASS_BEFORE_CLOSE);
      };
    } else {
      // 開く（details.open = true は click のデフォルト動作で行われる）
      content.animate(
        { height: ["0px", `${content.scrollHeight}px`] },
        { duration: DURATION, easing: EASING }
      );
    }
  });
});
```

### 実装例の補足

- `details`, `summary` 要素を使用します。これにより、開閉の機能をブラウザのデフォルト動作に任せることができ、アクセシビリティの要件を満たしやすくなります。
  - `summary` 要素をキーボードでフォーカスでき、`Enter` や `Space` で開閉できます。
  - 開閉状態がスクリーンリーダーに伝わります。
  - `details` 要素内のテキストは、`display: none;`で非表示にされた要素とは異なり、隠れている状態でもページ内検索の対象となります。
    - これは [`hidden="until-found"`](https://developer.mozilla.org/ja/docs/Web/HTML/Global_attributes/hidden#hidden_until_found_%E7%8A%B6%E6%85%8B) と同様の挙動です。
- アニメーションについては詳細後述。

### ビジュアルデザイン

- 角丸にする際の調整
  - 開閉前後どちらの状態でも角丸の見た目になるように、`details` 要素全体に `border-radius` を指定します。子要素に設定した背景色がはみ出さないよう、`overflow: hidden/clip;` も指定します。
  - その際、フォーカスリングが隠れてしまわないように注意してください。 `summary` 要素の領域からはみ出たフォーカスリングが隠れてしまい見えにくくなる場合があります。
  - そのため、この例では、`outline-offset` を負の値に設定することで内側にずらして隠れないように調整しています。
  - この他にも、フォーカスされているとき (`:focus-visible`) に特有のスタイルを当てることも選択肢です。フォーカスされていることがわかるようにしてください。

### アニメーション

開閉時にアニメーションをつけることで、新しいコンテンツが表示されたことを視覚的に認識しやすくできます。
開閉アニメーションの実装方法は、対象ブラウザによって変わります。

- `interpolate-size` が使えるブラウザだけが対象なら → **CSS 版**（JS 不要）
- 古いブラウザも対象なら → **JS 版**（Web Animations API）

#### CSS 版（2026 年 6 月現在は限られたブラウザでのみ動作）

:::message

この手法で使用している `::details-content` と `transition-behavior` はすでに主要ブラウザで利用できますが、**`interpolate-size` が Chromium 系ブラウザでしか対応していない**ため、この CSS のみの手法が動作するのは現状 Chromium 系に限られます。

ただし、これらに対応していないブラウザでは単にアニメーションしないだけであり、開閉自体は問題なく行えるので、それを理解した上でこの手法を選択するのもよいでしょう。

:::

近年になって対応が進んだ 3 つの機能を使うことで、CSS のみでアニメーションを実現できます。

- **`interpolate-size`**：`allow-keywords` を指定すると、`height: auto` などのキーワードへのアニメーションが許可され、`0` → `auto` の高さ変化を補間できるようになります。
  - 高さのアニメーションをつけない場合はつかわなくても問題ありません。
- **`transition-behavior`**：`allow-discrete` を指定すると、`content-visibility` のような離散プロパティ（中間値を持たず、値が切り替わるだけのプロパティ）も `transition` の対象にできます。`::details-content` は閉じているとき `content-visibility: hidden` になるため、これがないと閉じるときにアニメーションできず一瞬で非表示にされてしまいます。
- **`::details-content`**：`details` の開閉するコンテンツ部分を表す擬似要素です。ラッパー要素を自分で用意しなくても、開閉するコンテンツに直接 `height` や `transition` を指定できます。

対応状況は次のとおりです（2026 年 6 月時点）。最新の状況や対象ブラウザのバージョンは、各リンクを参照してください。

| 機能                                      | 対応状況                                                               | MDN                                                                                        | Can I use                                               |
| ----------------------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ | ------------------------------------------------------- |
| **`interpolate-size`**                    | Limited availability（Chromium 系のみ対応、Firefox / Safari は未対応） | [MDN](https://developer.mozilla.org/ja/docs/Web/CSS/interpolate-size)                      | [Can I use](https://caniuse.com/wf-interpolate-size)    |
| **`transition-behavior: allow-discrete`** | Baseline 2024（2024 年 8 月に主要ブラウザで対応済み）                  | [MDN](https://developer.mozilla.org/ja/docs/Web/CSS/transition-behavior)                   | [Can I use](https://caniuse.com/wf-transition-behavior) |
| **`::details-content`**                   | Baseline 2025（2025 年 9 月に主要ブラウザで対応済み）                  | [MDN](https://developer.mozilla.org/ja/docs/Web/CSS/Reference/Selectors/::details-content) | [Can I use](https://caniuse.com/wf-details-content)     |

```css:CSS
/* CSS (interpolate-size) を用いてアニメーションさせる */
details {
  /* allow-keywords によって、height: auto などキーワードへのアニメーションが適用される */
  interpolate-size: allow-keywords;

  /* ユーザーがアニメーションを無効にしていない場合のみアニメーションを行う */
  @media (prefers-reduced-motion: no-preference) {
    &::details-content {
      /* ::details-content に対する transition を使用する場合、overflow: clip ではみ出した部分を隠さないとアニメーション中に中身のテキストが見えてしまいます。 */
      overflow: clip;

      /* block-size は height に置き換えても大丈夫です */
      /* height と違い block-size は書字方向に依存したプロパティのため、縦書きでもうまく動作します */
      block-size: 0;

      transition-property: block-size, content-visibility;
      transition-duration: 0.4s;
      /* allow-discrete キーワードによって、content-visibility のような離散プロパティもアニメーションさせることができる */
      transition-behavior: allow-discrete;
    }

    &[open]::details-content {
      block-size: auto;
    }
  }
}
```

#### フォールバック版：JavaScript (Web Animations API) で height をアニメーション

前述の機能が使えないブラウザも対象にする場合は、JavaScript で高さをアニメーションさせます。ライブラリを使わずとも、ブラウザネイティブの [Web Animations API](https://developer.mozilla.org/ja/docs/Web/API/Web_Animations_API)（`Element.animate()`）で `height` をアニメーションさせることで実装できます。
実装は上の「実装例」のコード（HTML / CSS / JavaScript）を参照してください。

- アニメーションを設定する際、高さを変化させる要素と`padding`をつける要素はそれぞれ別になるように注意してください。高さの計算時に`padding`が考慮されません。

## ありがちな失敗例

### 開閉ボタンがボタンでない（キーボードでフォーカスできない）

見出しなどの要素に直接クリックイベントを付けて開閉する実装です。

```html:❌️ NG
<div class="trigger" onclick="toggle()">開閉する見出し</div>
```

`div` はボタンではないため、`Tab` でフォーカスできず、`Enter` / `Space` でも開閉できません。スクリーンリーダーにも「ボタンであること」「開いているか閉じているか」が伝わりません。

`details` / `summary` を使えば、キーボード操作も役割 (role) も開閉状態の読み上げもブラウザが自動で満たします（前述の実装例を参照）。

### overflow: hidden; でフォーカスリングが隠れてしまう

角丸 (`border-radius`) から子要素の背景色がはみ出さないように、`overflow: hidden;` を指定した場合に起きます。

```css:❌️ NG
.details {
  border-radius: 0.5rem;
  overflow: hidden;
}
```

`summary` にフォーカスしたとき、フォーカスリングが角丸の境界で切り取られて見えなくなります。`overflow: clip;` に変えても outline は切り取られるため、`overflow` の種類を変えるだけでは直りません。

`outline-offset` を負の値にしてリングを内側にずらすと、切り取られずに表示できます（前述の実装例を参照）。

### アニメーションが無効にできない

開閉アニメーションを `transition` だけで指定し、動きを減らす設定を考慮していない実装です。

```css:❌️ NG
::details-content {
  transition: block-size 0.4s;
}
```

「動きを減らす」設定（OS やブラウザの設定）を有効にしている人にもアニメーションが再生されます。動きで体調を崩す人（前庭障害など）には負担になります。

`prefers-reduced-motion` メディアクエリで、動きを減らしたい人にはアニメーションを無効化します（前述の実装例を参照）。

### ページ内検索で非表示になっているテキストが検索されない

`display: none;` で中身を出し入れする自作のアコーディオンで起きます。

```css:❌️ NG
.details__content {
  display: none;
}
.details.open .details__content {
  display: block;
}
```

閉じた状態では中身が `display: none;` で消えているため、`Ctrl` / `⌘` + `F` のページ内検索にヒットしません。情報がそこにあっても「無い」と判断されてしまいます。

`details` / `summary` なら、閉じていてもページ内検索の対象になり、ヒットすると自動的に開きます（前述の実装例を参照）。任意の要素で同じ挙動を得たい場合は [`hidden="until-found"`](https://developer.mozilla.org/ja/docs/Web/HTML/Reference/Global_attributes/hidden#hidden_until_found_%E7%8A%B6%E6%85%8B) を使います。
