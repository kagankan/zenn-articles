## はじめに

Claude Code や Codex CLI、Cursor のエージェントモードなどのコーディングエージェントはコーディングタスクに特化したものですが、それ以外の用途にも使うのも便利です。
テキストを処理する、という意味ではコーディングもその他の作業もあまり変わらないですから。

以下のような性質のタスクが使いどころです。

- テキストファイルの読み書き作業を行う
- （決まった処理ではなく）LLM を使った判断が必要
- 対話的な操作や、自然言語による曖昧な指示がしたい

[Spec Kit](https://github.com/github/spec-kit) のような仕様設計や、コードレビューのような開発に関連したタスクももちろん便利ですが、もっとコーディングに関係ないタスクでも、

- 資格勉強の確認テスト
- ブログ記事執筆
- プレゼン資料作成

などに活用できます。

この記事では、これらの用途でコーディングエージェントを活用するメリットやテクニックを紹介します。

## なぜコーディングエージェントを使うと便利なのか

コーディングタスクでないなら、ChatGPT などより汎用の AI サービスを使ってもいいかもしれませんが、あえてコーディングエージェントを使うことのメリットもあります。

### 普段使っている

なんといってもこれです。
普段の開発でコーディングエージェントを使っている場合、同じツールで共通化できるので、スイッチングコストを低くできます。
わざわざ別のツールを使うのは面倒です。

### ファイルを読み書きできる、しかも複数ファイルを自由に配置できる

ChatGPT などでもファイルの読み込みはできるし、書き込みも Canvas 機能でできます。
とはいえ、複数ファイルを同時に扱うのは難しいです。

コーディングエージェントなら、プロジェクトディレクトリに必要なファイルを自由に配置し、読み書きできます。

### バージョン管理ができる

Git で管理すれば、バージョン管理できるのも大きなメリットです。

- 変更履歴を追跡できる
- 過去の状態に戻せる
- ブランチを切って試行錯誤できる

### 複数人で共有しやすい

Git リポジトリとして管理すると、複数人での共有が簡単です。

- プロンプト（カスタムコマンドなど）の変更をプルリクエストでレビューできる
- 個人用途でも、複数デバイス間での同期が楽

### モデルの切り替えがしやすい

Cursor を使えば、タスクに応じてモデルを切り替えることができます。
簡単なタスクには軽量なモデル、複雑なタスクには高性能なモデルを使い分けられます。

### 必要に応じて、MCPや各種CLIツールを使わせることができる

音楽・動画ファイルの処理のために `ffmpeg` を使ったり、自前で書いたシェルスクリプトを動かしたりすることもできます。
これにより、ファイル処理など、複雑なタスクも任せることができます。

## 効率的に活用するテクニック

### カスタムコマンドを作成する

多くのコーディングエージェントには、カスタムスラッシュコマンドを定義する機能があります。
これを使えば、あらかじめ定義した手順に従ってタスクを行わせることができます。

Claude Code であれば、 `.claude/commands` に、Cursor であれば `.cursor/commands` にマークダウンファイルで定義します。

https://code.claude.com/docs/en/slash-commands#custom-slash-commands

https://cursor.com/ja/docs/agent/chat/commands

https://developers.openai.com/codex/guides/slash-commands/

### コマンド自体も AI に書かせる

前述のスラッシュコマンドを作成する際、スラッシュコマンド自体も AI に書かせることで効率化できます。

1. ざっくりとした指示だけ AI に伝える
2. AI がコマンドの詳細を書いてくれる
3. 必要に応じて微調整する

手書きしても同じだろうと思うかもしれませんが、自分が具体的に書かなかった部分に関してもモデルが詳細化して、かつ手順を整理して書いてくれるのでいい感じです。
一から書くよりも圧倒的に楽なので、ぜひ試してみてください。

### シンボリックリンクで両対応

複数のツールでカスタムコマンドや指示を共通化したい場合、シンボリックリンクを使えば設定ファイルを共有できます。

例えば以下のようなシンボリックリンクを定義しています。

- `.cursor/commands` → `.claude/commands` へのシンボリックリンク
- `AGENTS.md` → `CLAUDE.md` へのシンボリックリンク

### なんでもかんでもAIに任せず、決定論的に処理できるところは決定論的に処理する

ここまでは AI を使うと便利だ、という話をしてきましたが、なんでもかんでも AI に任せるとよいということはありません。
AI による処理・判断はあくまで確率論的な動作であり、確実性を持ちません。また、多くの場合時間もかかるし金銭的なコストもかかります。
なので、決定論的に（たとえばシェルスクリプトなどで）処理できるところは決定論的に、決まった手順で処理させるのがベターです。

たとえば、Spec Kit における動作でも、「ブランチを作成する」「ファイルをコピーする」といった処理はシェルスクリプトで記述されており、カスタムコマンドからはそのシェルスクリプトを実行するようになっています。
このように、役割分担をしていくことで効率化できます。

## おすすめの活用シーン

具体的な活用例を紹介します（余力があれば、それぞれ別記事で詳しく紹介予定）。

### 勉強編

AI に問題を出してもらうことで、勉強効率を高めることができます。
試験範囲の教材をテキストとして与え、それを元に問題を生成させるといったこともできます。

例えば、「白紙勉強法」と呼ばれるような、覚えたことを何も見ずに思い出すトレーニングにも使うことができます。

1. 覚えたい内容のテキストをテキストファイルとして与える
2. AI にテキストを読み込ませ、覚えるべき要点をチェックリストとして抽出させる
3. テストファイルとチェックリストを元にして、白紙勉強法を行わせる
   1. ユーザーが覚えている内容を入力（音声入力もよし）
   2. それが正しい内容か、チェックリストに含まれる内容かどうかを AI に判断させる
4. 思い出せなかった内容を AI に教えてもらう

https://toyokeizai.net/articles/-/852740?display=b

### 記事執筆編

Zenn の記事執筆にも活用できます。

- 骨子から肉付けさせ、ドラフトを作成してもらう
  - 作成させるための執筆スタイルガイドも、既存記事から AI に要約させる
- 記事をレビューしてもらう・誤字を指摘してもらう
- slug, emoji, topics などのメタデータを考えてもらう

参考：
https://zenn.dev/yoshiko/articles/my-ai-writing

### 資料作成編

プレゼン資料やドキュメント作成にも活用できます。
弊社では教材制作にも活用しています。

- 要件から、構成を提案させる
- [Marp](https://marp.app/) を使って、Markdown ファイルでスライドを作成する

## おわりに

AI コーディングエージェントは、コーディング以外でも意外と活用のしどころがあります。
ぜひ、ご自身の用途に合わせて試してみてください！
