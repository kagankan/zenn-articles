## 作ったもの

今回紹介するのは、社内アイデアソンを行った際にそのときのチームで作成した実験的な学習の仕組みです。
教材のテキストを元に、AI が対話形式で理解度を確認してくれる仕組みとして、「白紙勉強法」と呼ばれる学習手法を参考に実装してみました。

:::message

**白紙勉強法とは**
覚えたい内容を何も見ずに思い出して書き出すトレーニング手法。アクティブ・リコール（能動的想起）とも呼ばれ、記憶の定着に効果的だと言われている。

:::


### 動作の流れ

コマンドを実行すると、「学んだ内容を説明してください」と言われるので、頑張って思い出して説明する：

![/exam コマンドの実行時のスクリーンショット。「text.txtで学んだ内容について、自由に説明してください」というメッセージが表示されている。](/images/ai-exam-chat-with-agent/2025-12-10-04-17-10.png)

覚えている内容を説明すると、AI がその内容を評価し、チェックリストを埋めていく：

![](/images/ai-exam-chat-with-agent/2025-12-10-04-20-05.png)

対話を繰り返し、これ以上思い出せなくなったら、それ以降はチェックリストの内容に沿って 1 問ずつ問題が出される：

![](/images/ai-exam-chat-with-agent/2025-12-10-04-48-21.png)

## リポジトリ構成

非常にシンプルな構成です。

```
.
├── .claude/
│   └── commands/
│       └── exam.md          # カスタムスラッシュコマンド
└── contents/
    ├── text.txt             # 教材のテキスト
    └── checkList.md         # 理解度チェックリスト
```

## 動作の流れ

### 準備編

#### 教材のテキストを準備する

今回は、弊社 e ラーニングプラットフォームで提供している動画から、「ディープラーニング」を説明した動画の字幕テキストを使用しました。
これを `text.txt` に保存します。

#### チェックリストを作成する

教材のテキストから AI にチェックリストを生成させ、 `checkList.md` に保存します。

```txt:教材からチェックリストを作成させるプロンプト
'./contents/text.txt' この内容から理解度をチェックできるリストを作ってください
```

すると、以下のようなチェックリストが生成されます。

```markdown:checkList.md（一部抜粋）
# ディープラーニング・ニューラルネットワーク 理解度チェックリスト

## 基礎知識の確認

### 1. ディープラーニングの定義

- [ ] ディープラーニングがベースとしているアルゴリズムは何か説明できる
- [ ] ディープラーニングがどのような領域（データの種類）への適用を可能にしたか説明できる

### 2. ニューラルネットワークの構造

- [ ] ニューラルネットワークを構成する 3 つの層の名称を挙げられる
- [ ] 中間層が「隠れ層」とも呼ばれる理由を説明できる
- [ ] 入力層・出力層・中間層のうち、何層でも重ねることができるのはどれか答えられる
```

#### カスタムスラッシュコマンドを作成する

`.claude/commands/exam.md` に、テストの流れを記述し、AI コーディングエージェントが試験官として振る舞うようなカスタムコマンドを作成します。

このマークダウンファイルがそのまま AI へのプロンプトになります。
ちなみに、このカスタムコマンドもコーディングエージェントに書かせています。

```markdown:exam.md（一部抜粋）
### ステップ1：白紙勉強法（自由記述）

3. **text.txtを読み込む**
   - 学習内容を把握する

4. **自由に説明を促す**
   - 「それでは、text.txtで学んだ内容について、自由に説明してください。
     何も見ずに、覚えていることを自分の言葉で説明してみましょう。」と促す
   - 箇条書き、文章、図解など、どんな形式でも構わないことを伝える

5. **説明を評価する**
   - ユーザーの説明を丁寧に聞く
   - text.txtの内容と照らし合わせて、どの項目が理解できているか確認する
   - 理解できている項目を特定し、チェックリストにチェックを入れる
```

:::details exam.md 全文

```markdown
---
description: 学習内容の理解度を確認する対話型テスト
---

あなたは学習内容の理解度を確認する教育アシスタントです。以下の手順で進めてください：

## 前提条件
- ユーザーは './contents/text.txt' の内容を学習している
- 理解度チェックリストは './contents/checkList.md' にある

## 実行手順

### ステップ0：チェックリストのリセット

1. **checkList.mdを読み込む**
   - 現在のチェックリストの状態を確認する

2. **すべてのチェックボックスをリセットする**
   - すべての `- [x]` を `- [ ]` に変更する
   - Edit ツールを使用してファイルを更新する
   - 「チェックリストをリセットしました。新しいセッションを開始します。」と伝える

### ステップ1：白紙勉強法（自由記述）

3. **text.txtを読み込む**
   - 学習内容を把握する

4. **自由に説明を促す**
   - 「それでは、text.txtで学んだ内容について、自由に説明してください。何も見ずに、覚えていることを自分の言葉で説明してみましょう。」と促す
   - 箇条書き、文章、図解など、どんな形式でも構わないことを伝える

5. **説明を評価する**
   - ユーザーの説明を丁寧に聞く
   - text.txtの内容と照らし合わせて、どの項目が理解できているか確認する
   - 理解できている項目を特定し、チェックリストにチェックを入れる

6. **フィードバックを提供する**
   - 「よく理解できている点」を具体的に褒める
   - 「もう少し詳しく説明してほしい点」があれば追加質問する
   - 現在の進捗状況を表示する（例：「18項目中10項目カバーされています」）

### ステップ2：個別項目の確認

7. **未チェック項目を確認する**
   - 白紙勉強法でカバーされなかった項目を確認する
   - 未チェックの項目が複数ある場合は、上から順番に1つずつ確認する

8. **質問を行う**
   - 未チェックの項目について、1つずつ質問を行う
   - 質問は具体的かつ明確にする
   - 例：「ディープラーニングがベースとしているアルゴリズムは何ですか？」

9. **回答を評価する**
   - ユーザーの回答を慎重に評価する
   - 正解の場合：「正解です！」と伝え、該当項目にチェックを入れる
   - 不正解の場合：ヒントを与えるか、text.txtの該当箇所を示して復習を促す
   - 部分的に正解の場合：追加の質問をして理解度を確認する

10. **checkList.mdを更新する**
   - 理解できていると判断した項目は、チェックボックスを`- [x]`に変更する
   - Edit ツールを使用してファイルを更新する

11. **進捗を確認する**
   - 現在の進捗状況を表示する（例：「18項目中15項目完了」）
   - すべての項目にチェックが入るまで、ステップ8〜11を繰り返す

12. **完了時**
   - すべての項目にチェックが入ったら、「おめでとうございます！すべての項目を理解しました！」と伝える

## 注意事項
- 白紙勉強法では、ユーザーのペースで自由に説明させる
- 説明の途中で遮らず、最後まで聞く
- 個別項目の確認では、一度に1つの項目のみを確認する
- ユーザーが理解していない場合は、焦らせず丁寧に説明する
- text.txtの内容に基づいて評価する
- チェックリストは必ず最新の状態を保つ
- ユーザーが「わからない」「スキップしたい」と言った場合は、text.txtの該当箇所を示して復習を促す

それでは理解度確認を始めます。まずはcheckList.mdをリセットしてから、text.txtを読み込んで白紙勉強法を始めます。
```

:::

### 実行編

#### `/exam` コマンドを実行する

`.claude/commands/exam.md` に記述したカスタムコマンドを実行することで、学習内容の理解度を確認する対話型テストを開始します。

#### ステップ1：白紙勉強法

`/exam` コマンドを実行すると、まずチェックリストがリセットされます。
その後、「学んだ内容を自由に説明してください」と促されます。

ユーザーが覚えている内容を自由に説明すると、AI がその内容を評価し、該当する項目のチェックボックスに自動的にチェックを入れていきます。

例えば、「ディープラーニングはニューラルネットワークをベースにした手法で、画像やテキスト、音声領域への適用を可能にしました」と説明すると、関連する項目にチェックが入ります。

#### ステップ2：個別項目の確認

白紙勉強法でカバーできなかった項目について、AI が 1 つずつ質問してくれます。

```
AI: 「ニューラルネットワークを構成する 3 つの層の名称を挙げられますか？」
ユーザー: 「入力層、出力層、中間層です」
AI: 「正解です！」（checkList.md を更新）
```

すべての項目を確認できるまで、このやり取りが続きます。

## 実装上のポイント

### コーディングエージェントの活用のメリット

この仕組みを ChatGPT などの一般的な AI チャットや API を用いたアプリケーションではなく、コーディングエージェントで実装した理由はいくつかあります。

#### アプリケーションの開発なしに、AI とのチャット機能を素早く検証できる

このような機能を実装しようと思うと、UI を作って、API とのつなぎこみをして…といった手間がかかります。
一方、コーディングエージェントのカスタムコマンドを使えば、その手間を省略して、手っ取り早くチャット機能のみを実装・検証できます。

#### 複数ファイルの管理が楽

- `text.txt`（教材）
- `checkList.md`（チェックリスト）
- `exam.md`（実行手順）

これらのファイルを 1 つのプロジェクトとして管理できます。

#### 繰り返し実行が簡単

`/exam` と入力するだけで、同じ手順を何度でも実行できます。
毎回同じプロンプトを入力する手間がないですし、少し変更して試すことも容易です。

#### 教材の追加・編集が楽

テキストエディタで `text.txt` を編集すれば、すぐに新しい教材でテストできます。
別の科目や別のトピックにも簡単に応用できます。

## 学習方法としてのポイント

### メリット

#### 理解度の進捗がわかりやすい

チェックリストで管理しているので、進捗が視覚的にわかりやすくなります。

#### 2段階の学習方法の設計

白紙勉強法（自由記述）→ 個別質問という 2 段階構成にしています。

**白紙勉強法**は、自分の言葉で説明することで理解度が深まります。
従来の選択肢方式のテストとは異なり、当てずっぽうで選ぶだけでは正解できないので、能動的に自分の言葉で説明する必要があります。
こういった自由回答のテストは正誤判定が難しい形式でしたが、生成 AI を使用することで、柔軟に評価できます。

白紙勉強法に加えて**個別質問**を行うことで、あらかじめ用意した理解してほしい項目のチェックリストに漏れがないか確実に確認できます。
AI が適宜ヒントを出してくれたり、わからなかった場合も正解と該当箇所を示したりしてくれます。

#### "120点" の評価も可能

従来型のテストでは、用意した問題に対して正誤を判定するだけなので、当たり前ですが 100 点満点の評価になります。
しかし、人によってはテストで用意された問題以上に理解している場合があります。

従来の手法では評価できなかった、そういった人の知識を今回の手法では評価できるポテンシャルがあります。
あらかじめ用意したチェックリストの範囲を超えてユーザーが説明した場合でも、正解データである教材のテキストと照らし合わせることで、ユーザーの説明内容が正しいか評価できます。

### 課題

#### 回答の引き出し方が下手

質問の出し方が下手で、問題文の中に答えを書いてしまっていることがあるので、そのあたりは改善が必要です。
（例：「ディープラーニングは、画像やテキスト、音声といったどのような領域への機械学習適用を可能にしましたか？」「答え：画像、テキスト、音声」）

### 展望

#### 音声入力との組み合わせ

音声入力を活用することで、タイピングの手間を省き、よりスムーズに学習を進められます。
加えて、AI 側のメッセージも音声で出力することで、音声会話のみで学習を進めることも考えられます。

#### 難易度の調整

目的に応じてチェックリストの粒度を変えることで、難易度を調整できます。

- 初学者向け：大まかな項目のみ
- 中級者向け：詳細な項目も含める
- 上級者向け：応用的な質問も追加

#### 複数回実施の記録

毎回の実施結果を別ファイルに保存すれば、学習の進捗を追跡できます。
「前回より理解度が上がっている」といった分析も可能になります。

#### 弱点の可視化

間違えた項目や時間がかかった項目を記録すれば、自分の弱点が可視化でき、復習すべきポイントが明確になりそうです。

## まとめ

コーディングエージェントを用いた、対話型の理解度確認テストの実装方法を紹介しました。

一学習者としては、自分の理解度を可視化できるので、勉強の際にとても有用です。
また、アプリケーション開発者としては、UI を作成する手間を省略して、手っ取り早くチャット機能のみを実装できるので、素早く検証できることがメリットです。
