# タブ UI コンポーネント — 調査結果まとめ

## 1. WCAG 2.2 該当基準

### アコーディオンと共通する基準

- **達成基準 2.1.1 キーボード（レベル A）**
  - タブの選択・切り替え・タブリスト内の移動がすべてキーボードで操作可能であること。APG では矢印キーでタブ間移動、Tab キーでタブリストからパネルへ移動、Space/Enter でタブ選択（手動アクティベーションの場合）が求められる。
- **達成基準 2.4.7 フォーカスの可視化（レベル AA）**
  - 現在フォーカスされているタブにフォーカスインジケータが見えること。タブリスト内を矢印キーで移動するとき、どのタブにフォーカスがあるか視覚的にわかる必要がある。
- **達成基準 4.1.2 名前・役割・値（レベル A）**
  - `role="tab"` / `role="tablist"` / `role="tabpanel"` の付与、`aria-selected` による選択状態の通知、`aria-controls` によるタブとパネルの関連付けなど。
- **達成基準 2.3.3 インタラクションによるアニメーション（レベル AAA）**
  - タブ切り替え時にスライドやフェードなどのモーションアニメーションを使う場合、`prefers-reduced-motion` で無効にできるようにする。
- **達成基準 1.4.4 文字サイズの変更（レベル AA）** ／ **達成基準 1.4.12 テキストの間隔（レベル AA）** ／ **達成基準 1.4.10 リフロー（レベル AA）**
  - タブラベルのテキストが 200% まで拡大されてもはみ出さず読めること。画面幅 320px まで狭めても横スクロールなしで利用できること。

### タブ固有で追加される基準

- **達成基準 1.3.1 情報及び関係性（レベル A）**
  - タブとパネルの「関連付け」がプログラム的にも伝わること。`role="tablist"` / `role="tab"` / `role="tabpanel"` + `aria-controls` / `aria-labelledby` で構造を伝える。
- **達成基準 2.4.3 フォーカス順序（レベル A）**
  - アクティブタブ → Tab キー → パネル（またはパネル内の最初のフォーカス可能要素）という論理的な順序。非選択タブは `tabindex="-1"` で Tab 順序から外す。
- **達成基準 3.2.1 フォーカス時（レベル A）**
  - タブにフォーカスを当てただけでコンテキストの変化（ページ遷移等）を起こさないこと。パネル内のコンテンツ切り替えはコンテキストの変化に当たらない。
- **達成基準 1.4.11 非テキストのコントラスト（レベル AA）**
  - 選択中と未選択のタブを区別する視覚的変化（下線、背景色など）が隣接色に対して 3:1 以上のコントラスト比を持つこと。
- **達成基準 1.4.1 色の使用（レベル A）**
  - 選択中のタブを色だけで示さないこと。下線・太字・背景色の変化など色以外の手がかりも併用する。
- **達成基準 2.4.11 隠されないフォーカス（最低限）（レベル AA）** ※WCAG 2.2 新規
  - フォーカスされたタブがスティッキーヘッダーやオーバーレイで完全に隠れないこと。
- **達成基準 2.1.2 キーボードトラップなし（レベル A）**
  - タブリスト内やタブパネル内でフォーカスが閉じ込められないこと。Tab キーでタブリストから抜け出せること。

### 迷うが含めておく基準

- **達成基準 2.4.6 見出し及びラベル（レベル AA）** — タブラベルが対応するパネル内容を適切に表していること。
- **達成基準 3.2.4 一貫した識別性（レベル AA）** — サイト内で同じ機能のタブには一貫したラベルを使うこと。
- **達成基準 2.5.8 ターゲットのサイズ（最低限）（レベル AA）** ※WCAG 2.2 新規 — 各タブの操作領域が最低 24×24 CSS ピクセルであること。
- **達成基準 2.4.13 フォーカスの外観（レベル AAA）** — フォーカスインジケータの太さ 2CSS ピクセル以上、コントラスト比 3:1 以上。

## 2. 技術的詳細

### ARIA ロールと属性

| ロール | 付与する要素 | 説明 |
|--------|------------|------|
| `tablist` | タブの並びを囲むコンテナ | タブのグループであることを伝える |
| `tab` | 各タブ要素 | `tablist` の中に配置する |
| `tabpanel` | 各タブに対応するコンテンツ領域 | タブに紐づくコンテンツであることを伝える |

| 属性 | 付与先 | 値・説明 |
|------|--------|----------|
| `aria-labelledby` / `aria-label` | `tablist` | タブリスト全体のラベル |
| `aria-selected="true"` / `"false"` | `tab` | 選択状態 |
| `aria-controls` | `tab` | 対応する `tabpanel` の `id` を参照 |
| `tabindex="0"` / `"-1"` | `tab` | 選択中は `0`、非選択は `-1`（roving tabindex） |
| `aria-labelledby` | `tabpanel` | 対応する `tab` の `id` を参照 |
| `tabindex="0"` | `tabpanel` | パネル内にフォーカス可能要素がない場合に付与 |
| `aria-orientation` | `tablist` | 縦タブなら `"vertical"`。水平なら省略可 |

### キーボード操作仕様

| キー | 動作 |
|------|------|
| `Tab` | タブリストに入る（選択中タブにフォーカス）→ 次の `Tab` でパネルへ |
| `→` / `←` | 次/前のタブにフォーカス移動（末尾↔先頭ループ） |
| `↑` / `↓` | 縦タブの場合に前後移動 |
| `Enter` / `Space` | 手動アクティベーション時：フォーカス中のタブを選択 |
| `Home` / `End` | 最初/最後のタブにフォーカス（任意） |

### 自動 vs 手動アクティベーション

| | 自動 | 手動 |
|---|---|---|
| **動作** | 矢印キーでフォーカス移動と同時にタブ切り替え | 矢印キーではフォーカスのみ。Enter/Space で切り替え |
| **使い分け** | パネル表示に遅延がない場合 | ネットワーク通信等で遅延がある場合 |
| **APG 推奨** | 遅延がなければ自動を推奨（操作ステップが少ない） | — |

### ネイティブ HTML でのタブ実装

- **タブに相当するネイティブ HTML 要素は現時点で存在しない**（アコーディオンの `details`/`summary` のようなものがない）
- CSS のみの実装（ラジオボタン `:checked`、`:target` 方式）はアクセシビリティ上の問題が多く、APG のタブパターンを満たせない
- Open UI コミュニティが `<panelset>` 等のネイティブ要素を検討中だが、まだ仕様策定段階

### アコーディオンとの実装上の違い

| 観点 | アコーディオン | タブ |
|------|------------|------|
| **ネイティブ要素** | `details`/`summary` で JS 不要 | なし。ARIA + JS が必須 |
| **キーボード操作** | ブラウザが自動提供 | 自前で実装（roving tabindex、矢印キー） |
| **ロール・状態** | `details`/`summary` が自動提供 | 手動で ARIA 設定 |
| **ページ内検索** | 閉じていてもヒット | `display: none` パネルはヒットしない |
| **複数パネルの同時表示** | 可能 | 不可（常に 1 つだけ） |
| **フォーカス管理** | 各 `summary` が独立 | タブリスト内で roving tabindex |

### タブパネルのフォーカス管理

- パネル内にフォーカス可能要素がある場合: `tabindex="0"` 不要
- パネル内がテキストのみの場合: `tabindex="0"` を付与（Tab キーでパネルに移動できるようにする）
- APG 推奨: 一貫性のため全パネルに `tabindex="0"` を付けるのが安全

### 非表示パネルの扱い

- **推奨**: `hidden` 属性または `display: none` で非表示
- `aria-hidden="true"` 単体は不適切（視覚的にも隠す必要がある）

### 設計判断ポイント

- **タブ vs アコーディオンの使い分け**: タブは並列・同等のコンテンツ向き（2〜6 セクション程度）、アコーディオンは順序がある・長さが不均一なコンテンツ向き
- **タブが多い場合**: 横スクロール、オーバーフローメニュー
- **レスポンシブ**: デスクトップでタブ、モバイルでアコーディオンに切り替えるパターンが一般的
- **縦タブ**: `aria-orientation="vertical"` + ↑/↓キーでの移動

## 3. `focusgroup` 属性（Chrome 150〜）

Chrome 150（2026 年 6 月 30 日）で追加された `focusgroup` 属性は、**roving tabindex パターンを HTML 属性だけで宣言的に実現**するもの。タブ UI の矢印キー移動の JS 実装を置き換えられる。

### タブ UI での利用例

```html
<div role="tablist" focusgroup="horizontal wrap">
  <button role="tab" tabindex="0" aria-selected="true">Tab 1</button>
  <button role="tab" tabindex="-1" aria-selected="false">Tab 2</button>
  <button role="tab" tabindex="-1" aria-selected="false">Tab 3</button>
</div>
```

これだけで以下が自動的に動く:

- **左右矢印キー**でタブ間のフォーカス移動（`horizontal`）
- 末尾→先頭のループ（`wrap`）
- Tab キーでの出入りは既存の tabindex に従う（変更なし）

### roving tabindex との比較

| | roving tabindex（従来） | focusgroup（Chrome 150〜） |
|---|---|---|
| 矢印キー移動 | JS で keydown リスナー実装 | HTML 属性だけで実現 |
| tabindex の切り替え | JS で `0` / `-1` を動的に管理 | 初期値を設定すればブラウザが管理 |
| ループ（wrap） | JS で先頭/末尾の判定が必要 | `wrap` キーワードで宣言 |
| 縦タブ | JS で ↑/↓ を処理 | `vertical` に変えるだけ |

### 使えるキーワード

| キーワード | 効果 |
|-----------|------|
| `horizontal` | 左右矢印のみ（タブ UI の標準） |
| `vertical` | 上下矢印のみ（縦タブ用） |
| （省略） | 4 方向すべて |
| `wrap` | 端で折り返し |
| `grid` | 2 次元ナビゲーション |
| `extend` | 親の focusgroup に統合（ネスト構造用） |

### 対応状況と注意点

- **Chrome 150（2026 年 6 月 30 日）で Ship**。Firefox・Safari は未対応（2026 年 7 月時点）
- `focusgroup` が担うのは**矢印キーのフォーカス移動だけ**。以下は引き続き JS が必要:
  - `aria-selected` の切り替え
  - タブパネルの表示/非表示切り替え
  - 自動アクティベーション時のパネル切り替え
- 「最後にフォーカスされた要素の記憶」は現時点では未実装（将来 CSS Toggles との組み合わせで対応予定）

### チャプターでの扱い方の検討

アコーディオンの章で `interpolate-size`（Chromium 系のみ）を紹介したのと同じ構成で、「対象ブラウザで推奨実装が変わるパターン」として書ける:

- `focusgroup` が使えるブラウザのみ → focusgroup 版（矢印キー移動の JS 不要）
- 古いブラウザも対象 → JS 版（従来の roving tabindex）

ただし、`focusgroup` を使っても `aria-selected` 切り替えやパネル表示切り替えの JS は必要なので、アコーディオンの CSS 版ほど劇的な簡素化にはならない。「矢印キー移動の部分だけが宣言的になる」という位置づけ。

## 4. デザインシステム比較

| 項目 | W3C APG | MUI | Radix UI | React Aria (Adobe) | Inclusive Components |
|------|---------|-----|----------|--------------------|-----------------------|
| **デフォルトのアクティベーション** | 両方の例を提供 | 手動 | 自動 | 自動 | 手動 |
| **非アクティブパネル** | DOM に残し `hidden` | 実装依存 | アンマウント（`forceMount` で残せる） | inert（`shouldForceMount` で常時マウント可） | `hidden` 属性 |
| **縦タブ** | `aria-orientation` | サポート | `orientation` prop | `orientation` prop | 言及なし |
| **roving tabindex** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **レスポンシブ** | 言及なし | スクロール可能タブ | CSS に委ねる | `overflow-x: auto` | 言及なし |
